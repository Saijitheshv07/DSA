# [424. Longest Repeating Character Replacement](https://leetcode.com/problems/longest-repeating-character-replacement/description/)

- REVISIT
- They Key Idea here is use the maxFreq in the current window and not the actual maxFreq
- Even if maxFreq is stale, it only causes us to delay shrinking. Any window length we record must have been achievable earlier when maxFreq was accurate, so correctness is preserved.

### C++ Solution

```cpp
class Solution {
public:
    int characterReplacement(string s, int k) {
        map<char,int> mp;
        int left=0;
        int n=s.size();
        char r='*';
        int freq=0;
        int ans=0;
        for(int right=0;right<n;right++){
            mp[s[right]]++;
            freq = max(freq, mp[s[right]]);

                while(right-left+1-freq > k){
                    mp[s[left]]--;
                   left++;
                }
                ans = max(ans, right-left+1);
            }
             return ans;
        }  
};
```
### Java Solution

```java
class Solution {
    public int characterReplacement(String s, int k) {
        int[] freq = new int[26];
        int n = s.length();
        int f = 0;
        int i = 0;
        int ans = 0;
        for(int j = 0; j < n; j++){
            freq[s.charAt(j)-'A']++;
            f = Math.max(freq[s.charAt(j)-'A'], f);
            if(j-i+1 > f+k){
                freq[s.charAt(i)-'A']--;
                i++;
            }
            ans = Math.max(ans, j-i+1);
        }
        return ans;
    }
}
```
